# plan-review

실행 전 계획을 검토해 누락된 쟁점과 위험 요소를 찾아내는 스킬입니다.

## 목적

LLM 에이전트가 무언가를 실행하려 할 때, 계획 문서가 있든 대화 맥락만 있든 관계없이 **냉철한 서브에이전트가 계획을 검토**합니다. 비판적이고 객관적인 시각으로 계획의 허점, 모호한 지점, 숨은 위험을 지적하고, 모든 Critical·Major 이슈가 해결될 때까지 수정-재검토를 반복합니다.

## 사용 방법

에이전트가 자동으로 작동하거나, 직접 요청할 수 있습니다.

```
내 계획 검토해줘: ...
```

또는 실행 전에 에이전트가 스스로 트리거합니다.

## 작동 방식

```
Step 1: Plan 구조체 작성
  - 문서가 있으면 읽어서 변환
  - 대화 맥락만 있으면 추출하여 정리
  - 형식은 references/plan-structure.md 참조

Step 2: 서브에이전트(fork) 검토 의뢰
  - Plan 구조체만 전달 (원본 맥락은 전달하지 않음)
  - 5개 차원(완전성, 명확성, 실현 가능성, 위험 및 가정, 대안 고려) 검토

Step 3: 결과 처리 및 수정 반복
  - Critical/Major 없음 → PASS
  - Critical/Major 있음 → 수정 → 재검토 (최대 3회)

Step 4: 승인
```

## 파일 구조

```
plan-review/
├── README.md                          # 이 파일
├── SKILL.md                           # 에이전트용 워크플로우
└── references/
    ├── plan-structure.md              # Plan 구조체 명세
    ├── review-dimensions.md           # 5개 검토 차원 + 세부 기준
    └── reviewer-prompt.md             # fork 검토 프롬프트 템플릿
```

## 주의사항

- 이 스킬은 **계획의 검토만** 수행합니다. 실행은 별도로 진행해야 합니다.
- 서브에이전트는 전달받은 Plan 구조체만 보고 검토합니다. 원본 문서나 대화 맥락은 보지 않습니다.
- 단순 작업(Steps 2개 미만)은 자동 PASS됩니다.
- 3회 이상 BLOCK 시 Plan 자체의 근본적인 재검토가 필요할 수 있습니다.

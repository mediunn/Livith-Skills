# Livith Setlist Investigation

setlist.fm의 셋리스트 페이지가 주어졌을 때, 관련 내용을 조사하여 마크다운 파일로 정리하는 워크플로우.

에이전트는 가용한 도구(웹 fetch, bash 명령어, 파일 읽기/쓰기 등)를 자유롭게 활용하여 아래 단계를 수행한다.

## Step 1: setlist.fm 페이지 수집

setlist.fm URL에서 페이지 내용을 가져와 곡 목록과 공연 정보를 추출한다.

```
https://www.setlist.fm/setlist/{artist}/{year}/{venue}-{id}.html
```

페이지에서 다음 정보를 추출한다:
- 아티스트명
- 장소 (venue, city, state/country)
- 날짜 (**주의: setlist.fm은 보통 공연 당일만 표시하므로, 티켓 사이트·기사 등 외부 소스로 전체 공연 기간(시작일~종료일)을 반드시 별도 조사해야 한다**). 2일 이상이면 `~`로 표기, 예: `2026-05-16~17`.
- 셋타임 (시작/종료 시간)
- 전체 곡 목록 (커버곡 표시, 앵콜 구분)
- 게스트 (opener, 특별 게스트)
- 투어명 (있을 경우, setlist.fm 표기를 우선 따름)

**곡명 표기 규칙:**
- **일본어 곡명**: `원문 (로마자)` 형식 (예: `夜に駆ける (Yoru ni Kakeru)`)
- **한국어 곡명**: 원문 그대로 표기, 로마자 병기하지 않음
- **기타 비라틴 문자**: 일본어 규칙과 동일하게 처리
- 발음/로마자는 위키백과, 공식 유튜브 영상 제목, 또는 setlist.fm 영문 표기를 참고

## Step 2: 곡별 유튜브 링크 검색

각 곡의 공식 뮤직비디오, 리릭 비디오, 또는 official audio를 유튜브에서 검색한다.

**검색 전략:**
- **일본 아티스트:** 일본어 곡명(한자/가나)으로 검색하면 더 정확한 결과를 얻을 수 있다.
- **커버곡:** 검색어에 `cover` 및 원곡 아티스트명을 포함한다.

**결과 우선순위:**
1. **아티스트 공식 채널** — 공식 MV
2. **레이블 채널** (Innovative Leisure, Because Music, XL Recordings 등) — 공식 스트림
3. **공식 라이브 세션** (CBC Music, NPR Tiny Desk, KCRW, Audiotree 등)
4. 그 외 (팬 영상, 라이브 녹화)

검색된 영상의 채널명을 확인하여 우선순위를 판별한다.
공식 영상이 없으면 테이블에 `—`로 표시하고 비고에 `공식 영상 미확인`을 기재한다.
유튜브 영상 제목에서 원문 곡명을 추출하여 `원문 (로마자)` 형식으로 표시한다.

**검색 결과에 다음 메타데이터를 함께 수집한다:**
- YouTube ID
- 영상 제목 (full title)
- 채널명

이 메타데이터는 Step 2.5 검증 단계에서 사용된다.

## Step 2.5: 서브에이전트 전수 검증 (fork)

Step 2에서 수집한 전체 곡별 유튜브 링크와 메타데이터를 **서브에이전트(fork)에게 전달하여 전수 검증**을 수행한다.

서브에이전트는 각 YouTube 링크의 watch 페이지를 직접 fetch하여 다음을 확인한다:

### 2.5.1 영상 유형 분류

영상 제목과 description에서 아래 키워드를 감지하여 분류한다 (대소문자 무시, 해시태그 포함):

| 제목/설명에... | 분류 |
|---------------|------|
| `(Official Music Video)`, `(MV)`, `M/V` | **Official MV** |
| `(Lyric Video)`, `Lyrics`, `Lyric` | **Lyric Video** |
| `(Official Audio)`, `Official Audio`, `Audio Only` | **Official Audio** |
| `Live`, `(Live)`, `Live at` | **Official Live Video** |
| 위 키워드가 없고 공식 채널인 경우 | **Official Video** |

- 여러 키워드가 동시에 있는 경우 가장 구체적인 것으로 분류한다 (MV > Lyric > Audio > Live > 일반).
- `Live`가 `Live Session`, `Live at [장소]` 등 공식 라이브 세션인 경우에도 `Official Live Video`로 분류한다.

### 2.5.2 공식 채널 여부 확인

채널명과 아티스트명을 **퍼지 매칭**으로 비교한다:
- 대소문자, 공백, 특수문자(`/`, `-`, `_`, `(`, `)`, `Official` 등)를 무시한다.
- 아티스트명의 **핵심 키워드가 채널명에 모두 부분 포함(substring)** 되는지 확인한다.
- 예: `"Official HIGE DANdism"` ↔ `"Official髭男dism"` → `["hige", "dandism"]` 포함 → 일치 ✅
- 예: `"ヨルシカ / n-buna Official"` ↔ `"ヨルシカ"` → `["ヨルシカ"]` 포함 → 일치 ✅
- 핵심 키워드를 하나라도 포함하지 않으면 불일치 → `⚠️ 확인 필요`

### 2.5.3 ⚠️ 확인 필요 발동 조건

다음 중 하나라도 해당하면 비고에 `⚠️ 확인 필요`를 추가한다:

1. **채널명 불일치**: 퍼지 매칭 실패
2. **제목에 곡명 미포함**: 곡명의 핵심 키워드가 영상 제목에 포함되지 않음 (퍼지 매칭 동일 기준)
3. **커버/타아티스트**: 제목에 `cover`, `by`, `feat.` + 다른 아티스트명 포함 (원곡 아티스트가 아닌 경우)

`⚠️ 확인 필요`가 붙은 링크도 **삭제하지 않고 유지**한다. 비고에 경고와 함께 링크를 남겨 사용자가 판단할 수 있게 한다.

### 2.5.4 서브에이전트 출력

서브에이전트는 검증이 완료된 **최종 테이블 데이터**를 반환한다. 각 행은 다음 필드를 포함한다:
- `#` (순번)
- 곡명 (`원문 (로마자)` 형식)
- YouTube ID (없으면 `—`)
- 링크 (`https://youtu.be/{id}` 또는 `—`)
- 비고 (분류 + 필요시 `⚠️ 확인 필요`)

## Step 3: 공연 정보 조사

공연 정보(투어명, 장소, 날짜)를 다음 우선순위로 조사한다:

1. **티켓 예매 사이트**: `"{아티스트명} {도시} {날짜} ticket"` 등의 검색어로 일반 검색. 티켓 사이트는 공식 투어명과 장소 정보가 가장 정확하다.
2. **관련 기사**: 공연 리뷰, 투어 발표 기사 등에서 공연 컨텍스트 확인.
3. **setlist.fm / 공식 웹사이트**: 위에서 찾지 못한 경우.

- 공식 투어명이 확인되면 그대로 표기한다.
- 공식 투어명이 없으면 `"공식 투어명 없음"`으로 표기하고 임의로 만들지 않는다.
- **포스터**: 공식 포스터 이미지 URL을 검색하여 찾은 경우 추가. 없으면 생략.

### 3.1 여러 날짜 공연의 날짜 범위 조사 (필수)

**setlist.fm은 공연 당일만 표시하는 경우가 대부분이므로, 티켓 예매 사이트와 기사를 통해 전체 공연 기간을 반드시 조사한다.**

- 해당 공연이 여러 날짜에 걸쳐 진행된 경우(예: 2일 이상 동일 장소, 투어 내 연속 공연), **시작일과 종료일을 모두 조사**한다.
- `"{아티스트명} {장소} concert dates"`, `"{아티스트명} {연도} tour schedule"` 등의 검색어로 티켓 사이트·기사에서 확인.
- 날짜 표기는 `2026-05-16 ~ 2026-05-17` 형식으로 시작일과 종료일을 명시한다.
- 단일 공연으로 확인되면 해당 일자만 표기한다.
- setlist.fm에 `2026-05-16~17`처럼 축약된 경우에도 시작일·종료일을 풀어서 확인한 후 `2026-05-16 ~ 2026-05-17`로 표기한다.
- 티켓 사이트나 기사에서도 확인되지 않으면 setlist.fm 표기를 그대로 사용하고, 비고에 `⚠️ 확인 필요`를 남긴다.

## Step 4: 마크다운 문서 생성

서브에이전트가 반환한 검증 완료 데이터를 사용하여 다음 형식의 마크다운 파일을 생성한다. 출력 항목은 **공연명 / 일자 / 포스터 URL / 셋리스트**로 제한한다.

**문서 스타일 규칙:** 사용자가 읽기 편하도록 `**`(bold), `*`(italic) 등 마크다운 강조 문법은 출력물에 사용하지 않는다. 깔끔한 텍스트만으로 정보를 전달한다. (SKILL.md의 지시 사항 서식은 예외)

```markdown
# {셋리스트 제목 (아티스트 — Venue, City 날짜)}

공연: {투어명 또는 공연명}
일자: {날짜} (여러 날짜 공연이면 시작일 ~ 종료일, 예: `2026-05-16 ~ 2026-05-17`)
포스터: {URL} (없으면 생략)

| # | 곡명 | YouTube ID | 링크 | 비고 |
|---|------|-----------|------|------|
| 1 | {곡명} | {id} | https://youtu.be/{id} | {분류} |
```

**비고 컬럼 표기 예시:**
- `Official MV`
- `Official MV ⚠️ 확인 필요`
- `Lyric Video`
- `Official Audio`
- `Official Live Video`
- `Official Video`
- `공식 영상 미확인`

## Step 5: 출력 파일

파일명 규칙: `{아티스트}_{도시}_{연도}.md`
(예: `badbadnotgood_denver_2026.md`, `back_number_fukuroi_2026.md`)

아티스트명과 도시는 소문자, 공백은 언더스코어로 대체한다.
(ex. `BADBADNOTGOOD` → `badbadnotgood`, `Summit Music Hall, Denver` → `denver`)

파일은 현재 작업 디렉토리에 생성한다.
